Suunnitelma

Lukitusvaroitus

Ennen kuin agentti lukitsee ruudun, käyttäjää varoitetaan: agentti puhuu palvelimelta tulleen viestin (macOS say -tyyliin) ja näyttää laskurin, joka leijuu kaiken päällä mutta ei varasta fokusta eikä keskeytä peliä. Käynnissä olevaa Steam-peliä voi pelata varoitusajan loppuun asti. Kun laskuri päättyy, nykyinen lukitusoverlay ottaa ruudun haltuun.

Tausta ja tavoite

Nykyään admin lukitsee laitteen lock-komennolla, ja ruutu menee kerralla mustaksi ilman ennakkovaroitusta. Tavoite on antaa pelaajalle ennakkovaroitus äänellä ja näkyvällä laskurilla.

Valittu malli (vahvistettu): varoitus on osa lock-komentoa, ei erillinen komento. Yksi admin-klikkaus varoittaa ja lukitsee. Varoituslogiikka asuu klientillä, kuten lukitustarkistuskin.

Reunaehdot

  • Laskuri ei ole tavallinen ikkuna: läpiklikattava (WS_EX_TRANSPARENT), ei-aktivoituva (WS_EX_NOACTIVATE), topmost, layered. Peli pysyy aktiivisena ja pelattavana.
  • TTS-ääni on ensisijainen varoitus. Se kuuluu aina, myös exclusive-fullscreen-pelin päältä. Visuaalinen laskuri on lisä, joka näkyy borderless/windowed-peleissä (käytännön yleisin tapaus).
  • Suomi-ääni: puhutaan saatavilla olevalla äänellä; valitaan fi-FI-ääni jos asennettu, muuten fallback. README:hen ohje Microsoft Heidi -äänen asennukseen.
Tunnettu rajoite: aito exclusive-fullscreen DirectX-peli ei näytä mitään ulkoista overlayta päällään. Tätä ei ratkota injektoimalla (Steam/Discord -tyyli) koska se on tämän projektin ulkopuolella. Siksi ääni kantaa kaikkialle ja laskuri on lisä.

Datamalli: yksi uusi kenttä

Lisätään warning_seconds komentoon. Sama message toimii lukitusruudun tekstinä, puhuttavana varoituksena ja laskurin tekstinä, joten toista saraketta ei tarvita. warning_seconds > 0 tarkoittaa, että lukitusta edeltää varoitus. validCommand pysyy ennallaan (lock/unlock); varoitus ei ole oma komentotyyppi.

Server (server/internal/rootaika)

  1. store.go schema: lisää commands-tauluun warning_seconds INTEGER NOT NULL DEFAULT 0. migrate-funktioon ensureColumn(ctx, "commands", "warning_seconds", "INTEGER NOT NULL DEFAULT 0") (vrt. olemassa oleva message-sarakkeen ensureColumn).
  2. models.go: Command-structiin WarningSeconds int.
  3. store.go CreateCommand: lisää warningSeconds int -parametri ja INSERT. Päivitä molemmat SELECTit (PendingCommands, RecentCommands) lukemaan warning_seconds, ja scanCommands molemmat sarakemäärähaarat (7→8 ja 8→9 saraketta).
  4. handlers.go adminDeviceCommand: lue r.FormValue("warning_seconds") (parse int, clamp 0–600, oletus 0) ja välitä CreateCommand-kutsuun.
  5. handlers.go handleClientCommands: lisää "warning_seconds" JSON-mappiin.
  6. views.go lock-lomake: lisää numerokenttä viestikentän viereen, esim. <input name="warning_seconds" type="number" min="0" max="600" value="30">.

Windows-client (client-windows)

  1. internal/model/model.go: Command-structiin WarningSeconds int `json:"warning_seconds,omitempty"`.
  2. internal/config/config.go: Config-structiin LockWarningSeconds int `json:"lock_warning_seconds,omitempty"`.
  3. internal/serviceapp/app.go handleCommand: lock-haarassa aseta cfg.LockWarningSeconds = command.WarningSeconds, unlock-haarassa nollaa se. Lisää LockWarningSeconds agentStateResponse-structiin ja /agent/state-handleriin.
  4. cmd/rootaika-agent/main.go: serviceState-structiin LockWarningSeconds. Lisää siirtymän tunnistus paikallisella warnedForLock bool:lla. Loopissa ennen locker.SetLocked(true, ...):
    • jos state.Locked && !warnedForLock && state.LockWarningSeconds > 0: aja locker.Warn(ctx, state.LockMessage, state.LockWarningSeconds), aseta warnedForLock = true, vasta sitten SetLocked(true);
    • kun state.Locked == false: warnedForLock = false.
    Varoitus ajetaan niin että varoitusajan aikana tullut unlock peruuttaa sen (goroutine + ctx/peruutus, kun Locked muuttuu falseksi).
  5. internal/lock/controller_windows.go: uusi metodi Warn(ctx, message string, seconds int) error, joka tekee kaksi asiaa:
    • Puhuu viestin erillisellä PowerShellillä; teksti ROOTAIKA_SAY_TEXT-ympäristömuuttujassa (sama injektiosuoja kuin nykyinen ROOTAIKA_LOCK_MESSAGE). Yrittää valita fi-FI-äänen (GetInstalledVoices / SelectVoiceByHints), try/catch → fallback oletusääneen, sitten Speak.
    • Näyttää laskurin overlay-PowerShellillä kuten nykyinen lukitus, MUTTA ilman Activate()-timeria, ja asettaa ext-tyylit P/Invokella (SetWindowLong GWL_EXSTYLE): WS_EX_LAYERED | WS_EX_TRANSPARENT | WS_EX_NOACTIVATE | WS_EX_TOPMOST. Timer laskee sekunteja; ikkuna sulkeutuu kun laskuri = 0. Teksti envissä, sekunnit ROOTAIKA_WARN_SECONDS-envissä.
  6. internal/lock/controller_other.go: lisää Warn-stub (no-op), jotta Linux-buildi ja -testit kääntyvät, vastaten olemassa olevaa SetLocked-stubia.

Esimerkki: TTS-skripti (suomi + fallback)

Add-Type -AssemblyName System.Speech
$s = New-Object System.Speech.Synthesis.SpeechSynthesizer
try {
  $fi = $s.GetInstalledVoices() |
        Where-Object { $_.Enabled -and $_.VoiceInfo.Culture.Name -eq 'fi-FI' } |
        Select-Object -First 1
  if ($fi) { $s.SelectVoice($fi.VoiceInfo.Name) }
} catch { }
$s.Speak($env:ROOTAIKA_SAY_TEXT)

Teksti välitetään aina ympäristömuuttujassa, ei skriptiin interpoloituna, jotta viesti ei voi murtautua ulos PowerShell-merkkijonosta tai injektoida komentoja.

Uudelleenkäyttö (ei kirjoiteta uutta)

  • Komentoputki lock/unlock: CreateCommand → PendingCommands → JSON → model.Command → handleCommand → config → /agent/state → agentin loop. Varoitus kulkee samaa reittiä.
  • Injektiosuoja envin kautta: jo käytössä controller_windows.go.
  • Migraatiokuvio ensureColumn; snapshot/update-kuvio stateStore.

Todennus

  • Server-yksikkötestit (go test -race -cover ./...): warning_seconds persistoituu CreateCommand:ssa, näkyy PendingCommands- ja RecentCommands-tuloksissa, ja handleClientCommands palauttaa sen JSONissa.
  • Client-yksikkötestit: model.Command dekoodaa warning_seconds; handleCommand asettaa LockWarningSeconds. Windows-spesifit (Warn, overlay) eivät käänny Linuxilla → varmista että controller_other.go-stub kääntyy; oikea käytös todennetaan käsin Windowsilla (CLAUDE.md-konventio).
  • Manuaalinen Windows-testi: käynnistä server + service + agent; admin-UI:ssa viesti + warning_seconds=30 + Lock; varmista että agentti puhuu, laskuri näkyy borderless-Steam-pelin päällä, peli pysyy pelattavana, ja 30 s jälkeen musta lukitusoverlay tulee. Testaa unlock varoituksen aikana → lukitus peruuntuu ja laskuri katoaa. Testaa myös ilman suomi-ääntä (fallback ei kaadu).
  • Cross-compile: GOOS=windows go build ./... client-windows/:ssa.
