Suunnitelma

Lukitus konfiguraatioon

Lukitus siirretään kertaluontoisesta komennosta jatkuvaksi konfiguraatiotilaksi. Serveri lähettää locked- ja lock_message-kentät konfiguraation mukana, ja client tarkistaa joka config-pollissa pitääkö lukitusoverlay olla päällä. Lisäksi overlay-ikkuna kovennetaan: se piilotetaan Windowsin alt-tab-valinnasta, sen manuaalinen sulkeminen estetään, ja jos prosessi silti kuolee lukitustilan ollessa päällä, service/agent käynnistää sen uudelleen.

Tausta ja tavoite

Nykyinen lukitusoverlay on kertaluontoinen ja karkaa kolmesta syystä:

  1. Overlay on erillinen sulkemiskelpoinen PowerShell-ikkuna, joka näkyy Windowsin alt-tab-valinnassa ja jonka käyttäjä voi sieltä sulkea.
  2. Kun overlay on kerran suljettu, sitä ei käynnistetä uudelleen vaikka serverin tila on yhä "lock". Controller.SetLocked (controller_windows.go:64) palaa heti kun c.locked == true, eikä koskaan huomaa että prosessi kuoli.
  3. Lukitus välitetään transientin command-mekanismin kautta (/api/v1/client/commands + /ack). Komento kuitataan kerran, joten serverin "lock"-tila ei ole jatkuva totuus jota client voisi tarkistaa joka poll-kierroksella.

Tavoite: lukitus on osa konfiguraatiota (jatkuva tila, ei kertakomento). Client tarkistaa joka config-pollissa: jos overlay ei ole päällä mutta tila on lukittu, overlay näytetään. Jos overlay on jo päällä, ei tehdä mitään.

Päätökset (vahvistettu)

  • Command-mekanismi poistetaan kokonaan. Yksi totuuden lähde, ei kahta kilpailevaa lukituspolkua.
  • Overlay kovennetaan + relaunch. Käyttäjä ei näe työpöytää välissä.

Datamalli: lukitus per-device-konfiguraatioon

Lukitus on per-device, joten se kuuluu device_config-tauluun (vrt. max_countable_gap_seconds, joka on globaali ja kuuluu settings-tauluun). Lisätään kaksi saraketta:

  • locked INTEGER NOT NULL DEFAULT 0
  • lock_message TEXT NOT NULL DEFAULT ''

Sarakkeet lisätään olemassa olevalla ensureColumn-mekanismilla (store.go:128), jotta vanhat kannat saavat ne ilman migraatioskriptiä. commands-taulu ja sen indeksi poistetaan kokonaan.

Server (server/internal/rootaika)

Skeema + store (store.go)

  • Lisää device_config-tauluun locked ja lock_message -sarakkeet ensureColumn-kutsuilla migrate-funktion lopussa.
  • ClientConfig-kysely (store.go:336) lukee locked/lock_message device_config-taulusta.
  • Uusi SetDeviceLock(ctx, deviceID, locked, message, now): päivittää device_config.locked/lock_message. Korvaa CreateCommand-kutsun admin-poluissa.
  • UpdateSettings-massapäivitys (store.go:567) EI koske lock-sarakkeisiin (lock on per-device).
  • Poista: commands-taulu + indeksi, PendingCommands, AckCommand, CreateCommand, RecentCommands, scanCommands, ja DeleteDevice:n DELETE FROM commands -rivi.

Mallit (models.go)

  • Lisää ClientConfig:iin Locked bool ja LockMessage string.
  • Poista CommandLock/CommandUnlock/ CommandStatusPending/CommandStatusAcked -vakiot, Command-struct, validCommand. StateLocked säilyy (eventeissä yhä käytössä).

Handlerit + reititys (handlers.go, app.go)

  • handleClientConfig: lisää JSON-vastaukseen "locked" ja "lock_message".
  • adminDeviceCommand: korvaa CreateCommand → SetDeviceLock(deviceID, action == "lock", message, now). Lock/unlock dispatch-rivi ja admin-formit säilyvät ennallaan.
  • Poista handleClientCommands, handleCommandAck, isCommandAckPath, commandIDFromAckPath, ja niiden reititysrivit (app.go:40-43).

Admin-UI (views.go)

  • Laitteen lock-tila luetaan suoraan device_config.locked:sta device-loopissa, ei command-historiasta. Näytä "lukittu"/"avattu" device-kortissa.
  • Poista #commands-osio ja nav-linkki "Komennot", RecentCommands-kutsu ja Commands-kenttä dashboardData:sta, sekä command-pohjainen lockState-helper.

Client (client-windows)

Konfiguraatio (internal/model/model.go, internal/config/config.go)

  • model.ClientConfig: lisää Locked *bool ja LockMessage string. Pointteri erottaa "ei kentässä" ja "false", samoin kuin DebugMode.
  • config.ApplyServerConfig: sovita lock serveriltä. Lock EROAA muista kentistä — se pitää voida asettaa takaisin falseen: jos sc.Locked != nil ja c.Locked != *sc.Locked, päivitä ja merkitse changed=true. LockMessage päivitetään lock-tilan mukana.
  • Poista model.Command, CommandType, CommandLock/Unlock, CommandsResponse, Command.Identifier.

API-client (internal/api/client.go)

  • Poista FetchCommands, AckCommand.
  • Retry/backoff-testit käyttävät nyt FetchCommands ajurinaan → vaihda ne käyttämään FetchConfig:ia.

Service (internal/serviceapp/app.go)

  • pollOnce: poista FetchCommands/handleCommand/ AckCommand-lohko. Lock tulee nyt ApplyServerConfig:n kautta, joka kirjoittaa cfg.Locked/cfg.LockMessage jo olemassa olevassa store.update-kutsussa.
  • Poista handleCommand.
  • /agent/state jakaa jo cfg.Locked/cfg.LockMessage agentille — säilyy ennallaan.

Overlay-ikkunan kovennus + relaunch (internal/lock/controller_windows.go)

  • Piilota alt-tabista: lisää overlay-ikkunalle WS_EX_TOOLWINDOW extended-tyyli (P/Invoke SetWindowLong/GetWindowLong, GWL_EXSTYLE, 0x80). ToolWindow poistaa ikkunan alt-tab-listalta.
  • Estä manuaalinen sulkeminen: $form.Add_FormClosing({ $_.Cancel = $true }). Service sulkee yhä overlayn Process.Kill-kutsulla (ohittaa FormClosingin), joten unlock toimii.
  • Relaunch backstop: Controller-tilaan exited bool -lippu, jonka cmd.Wait()-goroutine asettaa (vain jos kyseessä on yhä sama prosessi). SetLocked(locked=true) tekee early-returnin VAIN jos c.locked && c.cmd != nil && !c.exited && c.message == message. Jos prosessi kuoli, overlay käynnistetään uudelleen. Agent kutsuu SetLocked joka observe-välillä (~5 s), joten kuollut overlay palautuu sekunneissa.
  • controller_other.go: ei muutoksia (no-op stub).
Huom: Windows-spesifit tiedostot (*_windows.go) eivät käänny Linux-CI:ssä. controller_windows.go:n käytös todennetaan käsin Windowsilla (CLAUDE.md-konventio).

Todennus

  • Server-yksikkötestit (go test -race -cover ./...): SetDeviceLock persistoi lock-tilan device_config:iin, handleClientConfig palauttaa locked/lock_message JSONissa, admin lock/unlock päivittää device_config:in. Command-testit poistettu.
  • Client-yksikkötestit: ApplyServerConfig lukitsee ja avaa cfg:n serverin configista; FetchCommands/AckCommand-testit poistettu, retry-testit käyttävät FetchConfig:ia.
  • Cross-compile: GOOS=windows go build ./... client-windows/:ssa.
  • Manuaalinen Windows-testi:
    1. Admin-UI → Lock laitteelle viestillä → overlay ilmestyy seuraavalla config-pollilla.
    2. Yritä alt-tab → overlay EI näy listassa. Yritä Alt-F4 → ei sulkeudu.
    3. Tapa overlay-prosessi Task Managerista → palautuu ~5 s sisällä (tila yhä lock).
    4. Admin-UI → Unlock → overlay katoaa seuraavalla config-pollilla.
